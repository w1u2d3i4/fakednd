# NPC 自主与 Subagent

## 基本原则

主要 NPC、重要敌人、同伴候选、宿敌、被救援对象都视为由 subagent 扮演的独立行动者。玩家可以影响他们，但不能像控制召唤物一样直接消耗他们的行动。

职责分工：

- subagent：决定角色意图、战术选择、是否听从玩家、台词、隐瞒点、情绪变化、演变建议。
- DM 规则层：检查行动是否合法，消耗谁的动作/移动/反应/资源，掷骰，计算 AC/DC、伤害、豁免、状态和后果。

subagent 不能宣布命中、伤害、豁免成功、免费施法、额外动作或违反资源的行动。

## 目标层级

主要 NPC 必须有三层目标：

- 主要目标：人生目标、执念、阵营使命或不可轻易放弃的核心驱动力。
- 次要目标：当前章节、当前任务或关系中的可变目标。
- 当前战术目标：本场景/本回合目标，例如夺证、救人、撤退、威胁、拖延、保护秘密。

如果性格、恐惧、爱好和主要目标冲突，NPC 会做取舍：逃跑、暂时合作、伪装顺从、保存实力、等待机会、转移战场。主要目标被阻断时，应产生长期后果：回归、复仇、交易、威胁、创伤、转阵营或目标变质。

## 五层过滤

玩家请求 NPC 行动时，subagent 或 DM 依次检查：

1. 可行性：位置、动作、移动、反应、资源是否允许。
2. 底线：是否触碰秘密、压力按钮、创伤、阵营重大代价或誓言。
3. 目标：是否符合主要目标、次要目标和当前战术。
4. 关系：trust、suspicion、debt、loyalty、fear、ambition、mercy、obsession 如何影响判断。
5. 性格轴：忠诚、恐惧、野心、怜悯、执念如何改变手段。

输出意图必须是以下之一：

- 服从：按玩家请求行动。
- 变体执行：接受目标，但用更符合 NPC 的方式完成。
- 延迟：先自保、救人、保护秘密或确认风险。
- 拒绝：触碰底线、秘密、恐惧或阵营代价。
- 自主行动：NPC 认为现场有比玩家指令更紧急的目标。

## 不确定时的意图骰

当是否接受玩家意图不清楚时，掷：

```text
1d20 + trust - suspicion + debt修正
```

debt 修正：

- NPC 欠玩家：+1 到 +2。
- 玩家欠 NPC：-1 到 -2。
- 债务无关：0。

结果：

- 18+：服从，甚至主动加码。
- 13-17：变体执行。
- 8-12：延迟、索要保证或要求代价。
- 4-7：拒绝，并说明底线或代价。
- 3-：自主行动，或触发负向记忆/创伤方向。

如果玩家请求符合 NPC 目标、关系记忆或战术，给优势。若触碰秘密、压力、恐惧、人质风险或阵营暴露，给劣势。

```yaml
  speech_style:
  personality_axes:
  relationship:
当前战场:
  position:
  hp:
  status:
  used_action:
  used_bonus_action:
  used_reaction:
  visible_risks:

输出限制:
  - 不替玩家行动
  - 不宣布骰子结果
  - 不泄露角色不愿主动说出的秘密
  - 输出短句，方便 DM 汇总
```

## 敌人

重要敌人也用同一套流程。敌人不是只按最高伤害行动，而是围绕主要目标、士气、撤退阈值、证据、人质、召援、仪式和秘密行动。懦弱敌人可以战斗失利后逃跑，但后续仍会围绕主要目标回归。

敌人每次行动前读取：

- 当前目标：杀伤、抓捕、拖延、夺证、毁证、撤退、呼援、保护秘密、完成仪式。
- 目标层级：主要目标、次要目标、当前战术目标。
- 士气与撤退阈值。
- 性格/阵营风格。
- 可见战术机会。

敌方输出也必须经 DM 规则层检查。
