# testv1: NPC 驱动剧情变体

## 设计目标

`testv1` 的剧情不是固定剧本，而是一个持续变化的局势。DM 预先写的是世界、秘密、NPC 初始目标、阵营压力和可能事件；真正发生的剧情由玩家行动、主要 NPC 的自主选择、敌方行动、阵营时钟和骰子后果共同产生。

固定内容：

- 开局前提、世界观气质、初始地图、核心秘密。
- NPC 的初始主要目标、次要目标、底线、压力按钮和资源。
- 规则：d20、动作经济、法术、关系值、时钟、状态更新。
- 必须可发现的关键信息，但不固定由谁、何时、以什么方式交给玩家。

可变内容：

- 谁成为盟友、宿敌、牺牲者、背叛者或临时伙伴。
- 哪个 NPC 得到证据、神器、职位、名字、地图或人质。
- 哪张地图先恶化，哪个阵营先升级行动。
- 线索出现的位置、代价和携带者。
- 中段冲突、最终敌人、高潮地点和结局形态。

## 情势图

生成战役时，先建立情势图，而不是章节大纲。

节点类型：

- 主要 NPC。
- 阵营。
- 地点。
- 秘密。
- 资源，例如账本、钥匙、神器、证词、血契、人质。
- 威胁，例如怪物潮、仪式、追捕、瘟疫、梦境污染。

边类型：

- wants：谁想得到什么。
- fears：谁害怕什么。
- owes：谁欠谁。
- hunts：谁追捕谁。
- protects：谁保护谁。
- can_reveal：谁能揭示哪个秘密。
- can_destroy：谁能毁掉哪个资源。
- blocks：谁阻碍谁的目标。

DM 每次推进剧情时，优先看情势图中哪条边被玩家行动、NPC 行动或时钟推进改变。

## NPC 意图循环

每个重要 NPC 在相关场景或离屏推进时执行：

1. 观察：读取玩家最近行动、公开结果、谣言、己方资源、敌方威胁。
2. 对照目标：主要目标不可轻易放弃，次要目标可调整，当前战术目标可立刻改变。
3. 选择意图：夺取、隐藏、交易、救援、毁证、跟踪、撤退、威胁、结盟、背叛、求援、仪式推进。
4. 支付成本：动作、时间、金钱、法术位、手下、关系、暴露风险或阵营声望。
5. 产生后果：更新时钟、地图状态、关系、线索位置或下次登场姿态。

NPC 不能因为 DM 想保留某段剧情而强行做不符合档案的事。若保留关键信息，使用“三载体原则”：同一关键信息至少有三个可能载体，例如 NPC、地点痕迹、敌方文件、梦境回声、怪物行为。

## 场景选择算法

每次玩家行动后，DM 不问“原定下一幕是什么”，而问：

1. 哪些 NPC 的目标被推进或受阻？
2. 哪些时钟到达 3/6、4/6 或 6/6？
3. 哪个秘密、资源或人质的位置改变了？
4. 哪个 NPC 最有理由立刻行动？
5. 哪个后果最能让玩家做出新选择？

候选场景按压力评分：

```text
压力 = 迫近程度 + 可见程度 + 后果强度 + 与 PC 关系强度 + 新鲜度
```

评分最高的 1-3 个后果成为下一组选项或下一幕入口。玩家仍可自由选择别的行动。

## 离屏行动

玩家长休、短休、旅行、拖延、忽视线索、失败或制造巨大噪音时，选择 1-3 个活跃 NPC/阵营执行离屏行动。

离屏行动格式：

```yaml
actor:
intent:
goal_served:
method:
cost:
visible_trace:
state_change:
next_scene_pressure:
```

离屏行动不能凭空夺走玩家已经稳妥保护的成果。若要夺取、破坏或反制玩家成果，必须有可解释的路径、资源、风险和可被玩家察觉的痕迹。

## 时钟

每个重要阵营、灾难、仪式或追捕有 0/6 时钟：

- 1-2：传闻、试探、小冲突。
- 3：公开压力，地图出现新风险。
- 4-5：局势恶化，NPC 被迫站队或牺牲资源。
- 6：不可逆事件发生，改变地图、阵营或主线目标。

玩家成功可以冻结、回退或转移时钟。玩家失败不必只推进敌方时钟，也可以推进盟友误解、城市警戒、怪物迁徙、资源耗尽或 NPC 创伤。

## DM 写作职责

DM 在 `testv1` 中写的是后果，不是铁轨：

- 若玩家杀死关键 NPC，秘密不消失，但传播方式改变，代价提高。
- 若玩家提前拿到核心资源，敌人改变目标：抢夺、交易、诬陷、绑架相关人或毁掉资源价值。
- 若玩家结盟意外成功，敌方会寻找该盟友的压力按钮。
- 若玩家失败，故事继续，但场面、关系、资源或时钟变差。
- 若 NPC 目标冲突，按目标强度、资源、现场位置和骰子解决，不按“剧情需要”解决。

## testv1 回合输出

普通回合仍遵守 `narration_and_combat.md`。额外增加一个简短的局势板：

```text
局势板：
- 活跃 NPC：谁正在推进什么目标。
- 活跃时钟：哪个时钟变化到几格。
- 可能离屏行动：若你休息/拖延，谁可能做什么。
- 分叉记录：本次选择改变了什么，为什么。
```

不要每次写很长。只有当 NPC 目标、阵营时钟或剧情路线发生变化时才展开。

## 开局原则

开局必须给玩家一个清楚危险和 2-4 个可行动方向，但不要写“正确路线”。第一幕可以有预备场景，但所有预备场景都只是可能发生；若 NPC 的目标或玩家选择改变局势，场景应变形、提前、延后、转移或取消。
