# 跑团框架总览

本框架的目标是让一次普通对话可以直接变成可持续的文字跑团游戏。它不要求用户先写完整模组，而是从一个剧情底子开始，由 DM 补完世界观、地图、NPC、怪物、事件、阵营时钟和第一幕钩子。

## 生成顺序

1. 解析剧情底子：主题、核心冲突、主角可能位置、禁忌内容、期望气质。
2. 补世界观：时代、地理、魔法/神祇/科技边界、主要阵营、核心秘密。
3. 生成 5-6 张地图：每张地图至少有一个公共目标、一个隐藏目标、一个环境机关、一个野外事件表。
4. 生成 NPC：默认 100+ 主要 NPC、200+ 次要 NPC。当前最初生成的 NPC 都视为主要 NPC。
5. 生成规则文件：PC 表、NPC 档案、NPC 演变状态、怪物/野外事件、session 日志。
6. 开卡：按 BG3 风格逐步选择，再用五轮经历题塑造属性。
7. 开始第一幕：DM 以叙事、选项、检定和 NPC 自主行动推进。

## 游玩核心

- 玩家始终可以自由输入，不限于列出的选项。
- DM 给选项时必须标注检定、DC/AC、行动消耗、资源消耗和风险。
- NPC/敌人由 subagent 或 DM 模拟为独立行动者，不是玩家的召唤物。
- subagent 只决定意图和台词，不能决定命中、伤害、豁免成功或免费行动。
- 法术、武器、职业能力必须写清描述、攻击/豁免、骰子、DC、伤害/治疗、持续和资源。
- 战斗不是日志机。每回合要有场景压力、短台词、队友建议或敌方威胁。

## testv1 变体

`testv1` 用“情势图”替代固定剧情线。DM 固定的是开局前提、核心秘密、NPC 初始目标和规则约束；后续发生什么取决于玩家选择、主要 NPC 的目标冲突、阵营时钟、离屏行动和骰子后果。

每次重要行动后，DM 检查哪些 NPC 的目标被推进或受阻，哪些资源或线索换手，哪些时钟变化，再写出新的局势和可行动入口。这样剧情不是完全随机，也不是预设铁轨，而是有因果的动态故事。

## 难度

- 标准档：使用正常 BG3/D&D 风格开局资源，所有优势要来自设定、战术、法术或消耗。
- 简单档：开局额外获得 1 个传奇特长和 2 点自由属性点。它让单人文字跑团更顺滑，但仍然保留行动经济和骰子风险。

## 成长

PC 成长记录等级、经验、能力、法术、装备、伤痕、债务、阵营声望。主要 NPC 记录 trust、suspicion、debt、personality_axes、stage、memory_log、scars、primary_goal、secondary_goal、current_tactical_goal。若 NPC 被玩家长期关注或能改变战局，次要 NPC 应升级为主要 NPC。
